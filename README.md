### Инструкция по выполнению домашнего задания

   1. Сделайте `fork` данного репозитория к себе в Github и переименуйте его по названию или номеру занятия, например, https://github.com/имя-вашего-репозитория/git-hw или  https://github.com/имя-вашего-репозитория/7-1-ansible-hw).
   2. Выполните клонирование данного репозитория к себе на ПК с помощью команды `git clone`.
   3. Выполните домашнее задание и заполните у себя локально этот файл README.md:
      - впишите вверху название занятия и вашу фамилию и имя
      - в каждом задании добавьте решение в требуемом виде (текст/код/скриншоты/ссылка)
      - для корректного добавления скриншотов воспользуйтесь [инструкцией "Как вставить скриншот в шаблон с решением](https://github.com/netology-code/sys-pattern-homework/blob/main/screen-instruction.md)
      - при оформлении используйте возможности языка разметки md (коротко об этом можно посмотреть в [инструкции  по MarkDown](https://github.com/netology-code/sys-pattern-homework/blob/main/md-instruction.md))
   4. После завершения работы над домашним заданием сделайте коммит (`git commit -m "comment"`) и отправьте его на Github (`git push origin`);
   5. Для проверки домашнего задания преподавателем в личном кабинете прикрепите и отправьте ссылку на решение в виде md-файла в вашем Github.
   6. Любые вопросы по выполнению заданий спрашивайте в чате учебной группы и/или в разделе “Вопросы по заданию” в личном кабинете.
   
Желаем успехов в выполнении домашнего задания!

Процесс выполнения<br>
Выполняя ДЗ сверяйтесь с процессом отражённым в записи лекции.<br>

Создайте Item который будет собирать информацию об загрузке CPU в процентах<br>
Создайте Item который будет собирать информацию об загрузке RAM в процентах<br>
Требования к результату<br>
 Прикрепите в файл README.md скриншот страницы шаблона с названием «Задание 1»<br>

# Задание 2
Добавьте в Zabbix два хоста и задайте им имена <фамилия и инициалы-1> и <фамилия и инициалы-2>. Например: ivanovii-1 и ivanovii-2.<br>

Процесс выполнения<br>
Выполняя ДЗ сверяйтесь с процессом отражённым в записи лекции.<br>
Установите Zabbix Agent на 2 виртмашины, одной из них может быть ваш Zabbix Server<br>
Добавьте Zabbix Server в список разрешенных серверов ваших Zabbix Agentов<br>
Добавьте Zabbix Agentов в раздел Configuration > Hosts вашего Zabbix Servera<br>
Прикрепите за каждым хостом шаблон Linux by Zabbix Agent<br>
Проверьте что в разделе Latest Data начали появляться данные с добавленных агентов<br>
Требования к результату<br>
 Результат данного задания сдавайте вместе с заданием 3<br>
 
# Задание 3
Привяжите созданный шаблон к двум хостам. Также привяжите к обоим хостам шаблон Linux by Zabbix Agent.<br>

Процесс выполнения<br>
Выполняя ДЗ сверяйтесь с процессом отражённым в записи лекции.<br>
Зайдите в настройки каждого хоста и в разделе Templates прикрепите к этому хосту ваш шаблон<br>
Так же к каждому хосту привяжите шаблон Linux by Zabbix Agent<br>
Проверьте что в раздел Latest Data начали поступать необходимые данные из вашего шаблона<br>
Требования к результату<br>
 Прикрепите в файл README.md скриншот страницы хостов, где будут видны привязки шаблонов с названиями «Задание 2-3». Хосты должны иметь зелёный статус подключения<br>
